🟠 HIGH Findings

H1 Twilio Webhook Endpoints Lack Signature Validation

Files: twilioRoutes.ts, appointmentCallController.ts

Risk: Webhook spoofing

Twilio routes accept both GET and POST without visible X-Twilio-Signature validation in the route middleware layer. The validation may exist inside controllers, but middleware-level enforcement is safer.

Fix: Add shared validateTwilioSignature middleware to all Twilio routes. Return 403 on invalid signatures.

H2 CORS credentials:false May Block Legitimate Cookie Auth

File: app.ts

CORS is credentials: false but frontend uses cookies for auth. If any client-side fetches need cookies, this could lead to an unsafe workaround.

Fix: Document the server-to-server assumption, or set credentials: true with explicit origin allowlist.

H3 JWT Cookie Security Flags Unverified

File: middleware.ts

The jlon_api_token cookie is read but the cookie-setting code wasn't in reviewed files. Without HttpOnly, Secure, and SameSite flags, tokens are vulnerable to XSS/MITM/CSRF.

Fix: Verify cookie flags: HttpOnly: true, Secure: true, SameSite: Strict

🟡 MEDIUM Findings

M1 Single JWT Secret, No Rotation

All tokens use one key. If compromised, attacker can forge ADMIN tokens.

Fix: Implement key rotation or switch to asymmetric keys (RS256).

M2 Impersonation Lacks Visible Audit Trail

Admin impersonation exists but no audit log route was found. Actions taken while impersonating may be untraceable.

Fix: Create immutable impersonation audit log table.

M3 Trust Proxy Chain Assumption

trust proxy: 1 assumes Cloudflare is always the first hop. Architecture changes could enable IP spoofing for rate limit bypass.

Mitigated: Custom resolveClientIp prefers cf-connecting-ip ✅

M4 OTP Rate Limit May Be Generous

30 requests per 10 min per IP. Safe for 6-digit codes, risky for 4-digit. No per-account limiting.

Fix: Ensure ≥6 digit OTPs + add per-account rate limits.

M5 Docker Runs as Root

No USER directive in Dockerfile. Container escape = root access.

Fix: Add non-root user to Dockerfile.

M6 SSH Key Mounted Into Container

Live compose mounts SSH keys (read-only) for git archives. Compromised container can read the key.

Fix: Use minimal-scope deploy keys. Consider separate archive container.

🟢 LOW Findings

L1 CSP allows unsafe-inline for styles

Common with Tailwind. Low risk but technically weakens XSS protection.

L2 GET accepted on Twilio webhooks

Twilio only POSTs. GET enables accidental triggers from browsers/crawlers.

L3 Zod validation errors expose field details

err.flatten() reveals internal schema to clients.

L4 1MB JSON body limit is generous

Most salon API requests are <10KB. Reduce to 256KB for general routes.

✅ What's Done Well
